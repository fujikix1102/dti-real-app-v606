# Likelihood Definition Binder — draft text V1

This panel is an audit-definition panel. It does not evaluate a likelihood, run CLASS, run MCMC, mount Planck clik, or compare posterior weights.

## Definition boundary

A likelihood result would require all of the following:

1. source-locked input data,
2. explicitly declared covariance or uncertainty model,
3. explicitly declared theoretical prediction path,
4. explicitly declared nuisance parameters and priors,
5. reproducible evaluation code,
6. preserved chain or grid output,
7. convergence / acceptance / burn-in / prior audit,
8. baseline comparison against an accepted reference model,
9. claim-boundary review before public or manuscript use.

The current viewer may display frozen identities, payload summaries, and audit-readiness status. It must not describe these displays as a posterior result, likelihood validation, Planck/CMB validation, DESI fit, physics proof, or Hubble-tension solution.
